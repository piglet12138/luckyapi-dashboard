**分析时间**: 2026-03-14
**分析目的**: 深度梳理复购率、付费转化率等核心指标的定义和合理性

## 一、复购率口径分析

### 1.1 当前定义

**计算公式**:
```sql
repurchase_rate = 多次充值用户数 / 累计付费用户数 * 100
```

**具体实现**:
```sql
-- 分子：充值次数 >= 2 的用户数
SELECT COUNT(DISTINCT user_id)
FROM dws_user_lifecycle
WHERE topup_count >= 2

-- 分母：累计付费用户数
SELECT COUNT(DISTINCT user_id)
FROM dwd_logs
WHERE log_type = 1
```

**当前数据**:
- 累计付费用户: 279人
- 多次充值用户: 110人
- 复购率: 110/279 = 39.43%

### 1.2 充值次数分布

| 充值次数 | 用户数 | 占比 |
|---------|--------|------|
| 1次 | 169人 | 60.6% |
| 2次 | 34人 | 12.2% |
| 3次 | 12人 | 4.3% |
| 4次 | 8人 | 2.9% |
| 5次及以上 | 56人 | 20.1% |

### 1.3 口径评估

**当前定义的问题**:

1. **时间维度缺失** ⚠️
   - 当前是历史累计数据
   - 无法反映近期复购趋势
   - 早期用户和新用户混在一起

2. **复购定义单一** ⚠️
   - 只区分"1次"和"≥2次"
   - 无法区分"偶尔复购"和"高频复购"
   - 2次充值和60次充值被同等对待

3. **时间窗口不明确** ⚠️
   - 没有限定复购的时间窗口
   - 1年前充值2次和1周内充值2次被同等对待

### 1.4 建议的复购率定义

**方案A：分层复购率（推荐）**

```sql
-- 低频复购用户（2-5次）
low_frequency_repurchase_rate = 充值2-5次用户 / 付费用户 * 100

-- 中频复购用户（6-20次）
mid_frequency_repurchase_rate = 充值6-20次用户 / 付费用户 * 100

-- 高频复购用户（>20次）
high_frequency_repurchase_rate = 充值>20次用户 / 付费用户 * 100
```

**方案B：时间窗口复购率**

```sql
-- 30日复购率：首次付费后30天内再次付费的比例
repurchase_rate_30d =
    首次付费后30天内再次付费的用户数 /
    首次付费距今超过30天的用户数 * 100

-- 90日复购率
repurchase_rate_90d =
    首次付费后90天内再次付费的用户数 /
    首次付费距今超过90天的用户数 * 100
```

**方案C：月度复购率（适合看趋势）**

```sql
-- 某月的复购率：该月有充值且历史已充值过的用户比例
monthly_repurchase_rate =
    该月充值且历史已充值过的用户数 /
    该月充值用户数 * 100
```

### 1.5 推荐方案

**建议采用"方案A + 方案C"组合**:

1. **分层复购率**（方案A）：
   - 用于用户价值分析
   - 识别高价值复购用户
   - 支持精细化运营

2. **月度复购率**（方案C）：
   - 用于趋势监控
   - 评估运营效果
   - 支持同比/环比分析

## 二、付费转化率口径分析

### 2.1 当前定义

**计算公式**:
```sql
conversion_rate = 当日付费用户数 / 当日活跃用户数 * 100
```

**当前数据（2026-03-14）**:
- 当日活跃用户: 64人
- 当日付费用户: 19人
- 转化率: 19/64 = 29.69%

### 2.2 口径评估

**当前定义的问题**:

1. **只看当日转化** ⚠️
   - 用户可能不会当天付费
   - 需要时间了解产品和建立信任
   - 当日转化率低估了真实转化能力

2. **无法评估新用户转化** ⚠️
   - 新用户和老用户混在一起
   - 无法单独评估新用户转化效果
   - 无法优化新用户引导流程

3. **缺少转化漏斗分析** ⚠️
   - 只有最终转化率
   - 不知道在哪个环节流失
   - 难以针对性优化

### 2.3 建议的付费转化率定义

**方案A：新用户转化率（推荐用于评估获客质量）**

```sql
-- 新用户当日转化率
new_user_d0_conversion_rate =
    当日新增且当日付费的用户数 /
    当日新增用户数 * 100

-- 新用户3日转化率（推荐）
new_user_d3_conversion_rate =
    首次使用后3天内付费的用户数 /
    首次使用距今超过3天的用户数 * 100

-- 新用户7日转化率
new_user_d7_conversion_rate =
    首次使用后7天内付费的用户数 /
    首次使用距今超过7天的用户数 * 100

-- 新用户30日转化率
new_user_d30_conversion_rate =
    首次使用后30天内付费的用户数 /
    首次使用距今超过30天的用户数 * 100
```

**方案B：整体转化率（推荐用于整体监控）**

```sql
-- 当日转化率（当前使用）
daily_conversion_rate =
    当日付费用户数 / 当日活跃用户数 * 100

-- 累计转化率
cumulative_conversion_rate =
    累计付费用户数 / 累计注册用户数 * 100
```

**方案C：转化漏斗（推荐用于优化）**

```sql
-- 注册 -> 激活（首次调用）
activation_rate = 有调用的用户 / 注册用户 * 100

-- 激活 -> 深度使用（>15次调用）
engagement_rate = 调用>15次的用户 / 有调用的用户 * 100

-- 深度使用 -> 付费
payment_rate = 付费用户 / 调用>15次的用户 * 100

-- 整体转化率 = activation_rate * engagement_rate * payment_rate
```

### 2.4 推荐方案

**建议采用"方案A + 方案B + 方案C"组合**:

1. **新用户3日转化率**（方案A）：
   - 核心指标，用于评估获客质量
   - 3天是合理的决策周期
   - 支持渠道ROI分析

2. **当日转化率**（方案B）：
   - 辅助指标，用于日常监控
   - 快速发现异常
   - 评估促销活动效果

3. **转化漏斗**（方案C）：
   - 分析工具，用于优化
   - 定位流失环节
   - 指导产品改进

### 2.5 实施建议

**优先级P0：新用户3日转化率**

```sql
-- 创建新表存储新用户转化数据
CREATE TABLE IF NOT EXISTS ads_new_user_conversion (
    cohort_date TEXT PRIMARY KEY,  -- 新用户首次使用日期
    new_users INTEGER,              -- 新增用户数
    d0_paying_users INTEGER,        -- 当日付费用户数
    d3_paying_users INTEGER,        -- 3日内付费用户数
    d7_paying_users INTEGER,        -- 7日内付费用户数
    d30_paying_users INTEGER,       -- 30日内付费用户数
    d0_conversion_rate REAL,        -- 当日转化率
    d3_conversion_rate REAL,        -- 3日转化率
    d7_conversion_rate REAL,        -- 7日转化率
    d30_conversion_rate REAL        -- 30日转化率
);

-- 计算逻辑
INSERT INTO ads_new_user_conversion
SELECT
    first_use_date as cohort_date,
    COUNT(*) as new_users,

    -- D0付费用户
    SUM(CASE WHEN first_topup_date = first_use_date THEN 1 ELSE 0 END) as d0_paying_users,

    -- D3付费用户
    SUM(CASE WHEN first_topup_date <= date(first_use_date, '+3 days') THEN 1 ELSE 0 END) as d3_paying_users,

    -- D7付费用户
    SUM(CASE WHEN first_topup_date <= date(first_use_date, '+7 days') THEN 1 ELSE 0 END) as d7_paying_users,

    -- D30付费用户
    SUM(CASE WHEN first_topup_date <= date(first_use_date, '+30 days') THEN 1 ELSE 0 END) as d30_paying_users,

    -- 转化率
    ROUND(CAST(SUM(CASE WHEN first_topup_date = first_use_date THEN 1 ELSE 0 END) AS FLOAT) / COUNT(*) * 100, 2) as d0_conversion_rate,
    ROUND(CAST(SUM(CASE WHEN first_topup_date <= date(first_use_date, '+3 days') THEN 1 ELSE 0 END) AS FLOAT) / COUNT(*) * 100, 2) as d3_conversion_rate,
    ROUND(CAST(SUM(CASE WHEN first_topup_date <= date(first_use_date, '+7 days') THEN 1 ELSE 0 END) AS FLOAT) / COUNT(*) * 100, 2) as d7_conversion_rate,
    ROUND(CAST(SUM(CASE WHEN first_topup_date <= date(first_use_date, '+30 days') THEN 1 ELSE 0 END) AS FLOAT) / COUNT(*) * 100, 2) as d30_conversion_rate

FROM dws_user_lifecycle
WHERE first_use_date IS NOT NULL
GROUP BY first_use_date
ORDER BY first_use_date;
```

## 三、渠道转化率图表数据不一致问题

### 3.1 问题描述

**现象**:
- 渠道转化率对比图表中，gemini-3-pro显示100%
- 但鼠标悬停时，tooltip显示44.23%
- 数据不一致

### 3.2 问题分析

**原因**:
- 图表数据来源：`dashboardData.channel_details`（已过滤new_users >= 10）
- Tooltip数据来源：通过`params[0].dataIndex`获取过滤后数组的元素
- **问题**：数据反转（reverse）后，dataIndex对应关系错乱

**代码分析**:
```javascript
// 第692-695行：数据过滤和排序
const data = dashboardData.channel_details
    .filter(d => d.new_users >= 10)  // 过滤
    .sort((a, b) => b.conversion_rate - a.conversion_rate)  // 排序
    .slice(0, 10);  // 取前10

// 第725行：Y轴数据反转
data: data.map(d => d.channel).reverse(),

// 第736行：柱状图数据反转
data: data.map(d => d.conversion_rate).reverse(),

// 第704行：Tooltip获取数据
const item = data[params[0].dataIndex];  // ❌ 错误：dataIndex是反转后的索引
```

**问题根源**:
- Y轴和柱状图数据都做了reverse()
- 但tooltip中使用的dataIndex是反转后的索引
- 导致`data[params[0].dataIndex]`获取到错误的元素

### 3.3 修复方案

**方案A：修正dataIndex（推荐）**

```javascript
formatter: function(params) {
    // 修正：反转索引
    const reversedIndex = data.length - 1 - params[0].dataIndex;
    const item = data[reversedIndex];
    return params[0].name + '<br/>' +
           '新增用户: ' + item.new_users + '<br/>' +
           '付费用户: ' + item.paying_users + '<br/>' +
           '转化率: ' + item.conversion_rate + '%';
}
```

**方案B：不反转数据**

```javascript
// 先反转数据，后续不再反转
const reversedData = data.reverse();

yAxis: {
    type: 'category',
    data: reversedData.map(d => d.channel),  // 不再reverse
    ...
},
series: [{
    ...
    data: reversedData.map(d => d.conversion_rate),  // 不再reverse
}]

formatter: function(params) {
    const item = reversedData[params[0].dataIndex];  // 直接使用
    ...
}
```

### 3.4 验证数据

```bash
# 查看gemini-3-pro相关渠道
sqlite3 newapi_warehouse.db "
SELECT
    first_model,
    SUM(new_user_count) as new_users,
    SUM(paying_user_count) as paying_users,
    ROUND(CAST(SUM(paying_user_count) AS FLOAT) / SUM(new_user_count) * 100, 2) as conversion_rate
FROM ads_channel_attribution
WHERE first_model LIKE '%gemini-3-pro%'
GROUP BY first_model
HAVING new_users >= 10
ORDER BY conversion_rate DESC
"

# 结果：
# gcp/gemini-3-pro-preview-thinking | 14 | 14 | 100.0  ✅
# gcp/gemini-3-pro-preview          | 7  | 5  | 71.43
```

**结论**:
- 100%的是`gcp/gemini-3-pro-preview-thinking`（14人全部付费）
- 44.23%可能是其他渠道的数据（因为索引错位）

## 五、推荐的核心指标体系

### 5.1 用户增长指标

| 指标 | 定义 | 用途 |
|------|------|------|
| DAU | 日活跃用户数 | 监控产品活跃度 |
| 新增用户 | 当日首次使用用户数 | 评估获客效果 |
| 7日留存率 | 7天前活跃用户今天仍活跃的比例 | 评估用户粘性 |

### 5.2 付费转化指标

| 指标 | 定义 | 用途 |
|------|------|------|
| **新用户D3转化率** | 新用户3天内付费比例 | 核心转化指标 |
| 当日转化率 | 当日活跃用户中付费比例 | 日常监控 |
| 累计转化率 | 累计付费用户/累计注册用户 | 整体转化水平 |

### 5.3 用户价值指标

| 指标 | 定义 | 用途 |
|------|------|------|
| ARPU | 人均收入 | 评估整体商业化 |
| ARPPU | 付费用户人均收入 | 评估付费用户价值 |
| **分层复购率** | 低频/中频/高频复购用户占比 | 用户价值分层 |
| 月度复购率 | 当月充值且历史已充值用户占比 | 复购趋势监控 |

### 5.4 渠道效果指标

| 指标 | 定义 | 用途 |
|------|------|------|
| 渠道新增用户 | 各渠道带来的新增用户数 | 评估渠道规模 |
| 渠道转化率 | 渠道总付费用户/总新增用户 | 评估渠道质量 |
| 渠道LTV | 渠道用户平均生命周期价值 | 评估渠道ROI |
