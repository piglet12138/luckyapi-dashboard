# 数仓表分类说明：增量表 vs 全量快照表

## 📊 一、增量表（Append-Only Tables）

### 定义
**增量表**是指只追加新数据，不更新历史记录的表。每次同步时，只插入新的数据行。

### 特点
- ✅ 数据只增不减
- ✅ 保留完整历史记录
- ✅ 查询时需要按时间过滤
- ✅ 适合行为日志类数据

### 增量表列表

| 表名 | 层级 | 数据来源 | 增量字段 | 说明 |
|------|------|---------|---------|------|
| `ods_logs` | ODS | API日志接口 | `created_at` | 所有日志记录（充值+消费+管理等） |
| `dwd_logs` | DWD | ods_logs清洗 | `log_date` | 清洗后的日志明细 |
| `dws_user_daily` | DWS | dwd_logs汇总 | `stat_date` | 用户每日使用统计 |
| `dws_model_daily` | DWS | dwd_logs汇总 | `stat_date` | 模型每日使用统计 |
| `dws_channel_daily` | DWS | dwd_logs汇总 | `stat_date` | 渠道每日使用统计 |
| `ads_daily_summary` | ADS | 各DWS表汇总 | `stat_date` | 平台每日汇总指标 |
| `ads_funnel_daily` | ADS | dwd_logs计算 | `stat_date` | 增长漏斗每日数据 |
| `ads_user_segment_daily` | ADS | dwd_logs计算 | `stat_date` | 用户分层每日统计 |
| `ads_activation_analysis` | ADS | dwd_logs计算 | `stat_date` | 激活点分析每日数据 |
| `ads_channel_attribution` | ADS | dwd_logs计算 | `stat_date` | 渠道归因每日数据 |

### 同步逻辑示例

```python
# 增量同步日志数据
def sync_logs_incremental():
    # 1. 获取上次同步的最大created_at
    last_max_created_at = get_last_max_created_at('ods_logs')
    
    # 2. 获取新数据（created_at > last_max_created_at）
    new_logs = api_client.get_logs(
        start_time=last_max_created_at + 1,
        end_time=int(datetime.now().timestamp())
    )
    
    # 3. 插入新数据（不更新已有数据）
    insert_into_ods_logs(new_logs)
    
    # 4. 更新同步记录
    update_sync_record('logs', max(new_logs['created_at']))
```

## 📸 二、全量快照表（Full Snapshot Tables）

### 定义
**全量快照表**是指每日全量同步，保留每日快照的表。每次同步时，覆盖当天的数据，但保留历史快照。

### 特点
- ✅ 保留每日快照
- ✅ 可以查询任意日期的数据状态
- ✅ 适合维度表和状态表
- ✅ 数据量相对较小

### 全量快照表列表

| 表名 | 层级 | 数据来源 | 快照字段 | 说明 |
|------|------|---------|---------|------|
| `ods_users` | ODS | API用户接口 | `sync_date` | 用户基础信息每日快照 |
| `ods_channels` | ODS | API渠道接口 | `sync_date` | 渠道配置每日快照 |
| `ods_groups` | ODS | API分组接口 | `sync_date` | 分组信息每日快照 |
| `dwd_users` | DWD | ods_users清洗 | `sync_date` | 用户明细每日快照 |

### 同步逻辑示例

```python
# 全量快照同步用户数据
def sync_users_snapshot():
    # 1. 获取所有用户数据
    all_users = api_client.get_all_users()
    
    # 2. 添加快照日期
    today = datetime.now().date()
    for user in all_users:
        user['sync_date'] = today
    
    # 3. 插入今日快照（保留历史快照）
    insert_into_ods_users(all_users)
    
    # 注意：不删除旧数据，保留历史快照
```

### 查询历史快照示例

```sql
-- 查询2026-03-01的用户状态
SELECT * FROM ods_users 
WHERE sync_date = '2026-03-01';

-- 查询用户状态变化
SELECT 
    user_id,
    username,
    quota,
    sync_date
FROM ods_users
WHERE user_id = 100
ORDER BY sync_date DESC;
```

## 🔄 三、混合表（Incremental Update Tables）

### 定义
**混合表**是指增量更新，但会更新已有记录的表。每次同步时，更新已有记录的累计指标，新增记录时插入。

### 特点
- ✅ 更新已有记录
- ✅ 保留累计指标
- ✅ 适合用户生命周期等需要累计计算的表

### 混合表列表

| 表名 | 层级 | 数据来源 | 更新策略 | 说明 |
|------|------|---------|---------|------|
| `dws_user_lifecycle` | DWS | dwd_logs计算 | `update_time` | 用户生命周期指标，每日更新累计值 |

### 同步逻辑示例

```python
# 增量更新用户生命周期
def update_user_lifecycle():
    # 1. 获取需要更新的用户（有新日志的用户）
    users_to_update = get_users_with_new_logs()
    
    # 2. 计算累计指标
    for user_id in users_to_update:
        # 计算累计调用次数
        total_calls = calculate_total_calls(user_id)
        
        # 计算累计充值金额
        total_topup = calculate_total_topup(user_id)
        
        # 更新或插入记录
        upsert_user_lifecycle(
            user_id=user_id,
            total_call_count=total_calls,
            total_topup_amount=total_topup,
            update_time=datetime.now()
        )
```

### 按数据特性分类

```

├── 日志数据（ods_logs, dwd_logs）

└── 分组信息（ods_groups）

状态数据（混合表）
└── 用户生命周期（dws_user_lifecycle）
```

## 🎯 五、设计原则

### 增量表设计原则
1. **主键唯一**: 使用业务主键（如log_id），避免重复插入
2. **时间分区**: 按日期分区，提高查询性能
3. **索引优化**: 在时间字段和常用查询字段上创建索引
4. **数据保留**: 考虑数据保留策略（如只保留1年数据）

### 全量快照表设计原则
1. **快照标识**: 使用 `sync_date` 字段标识快照日期
2. **联合主键**: 使用 `(业务主键, sync_date)` 作为联合主键
3. **历史保留**: 保留所有历史快照，便于追溯
4. **查询优化**: 在 `sync_date` 字段上创建索引

### 混合表设计原则
1. **更新标识**: 使用 `update_time` 字段记录最后更新时间
2. **幂等性**: 确保更新操作幂等，可重复执行
3. **增量计算**: 只计算新增数据，累加到已有记录
4. **性能优化**: 批量更新，避免逐条更新

## 📝 六、实施建议

### 首次同步
1. **增量表**: 全量插入历史数据（一次性）
2. **全量快照表**: 插入当前快照
3. **混合表**: 计算初始累计值

### 每日同步
1. **增量表**: 只同步新数据（created_at > 上次最大值）
2. **全量快照表**: 插入今日快照
3. **混合表**: 更新有变化的用户记录

### 数据清理
1. **增量表**: 定期清理旧数据（如保留1年）
2. **全量快照表**: 保留所有快照（数据量小）
3. **混合表**: 不清理（累计数据）

---

**总结**: 
- **增量表**适合行为日志，保留完整历史
- **全量快照表**适合维度数据，保留每日状态
- **混合表**适合累计指标，每日更新
