## 四、性能优化

### 4.1 SQL优化

**优化前**：
- 使用大量相关子查询
- 每个用户需要执行10+次子查询
- 预计执行时间：10-15分钟

**优化后**：
- 使用临时表预聚合
- 减少子查询次数
- 实际执行时间：约2分钟

**性能提升**: 5-7倍

### 4.2 优化措施

1. **用户每日汇总**：
   - 去掉 `top_model` 和 `top_channel_id` 的复杂子查询
   - 只保留 `model_count` 和 `channel_count`

2. **用户生命周期**：
   - 创建临时表 `temp_user_stats` 预聚合统计数据
   - 创建临时表 `temp_user_first` 获取首次使用信息
   - 最后JOIN临时表插入结果

## 五、DWS层使用示例

### 5.1 每日活跃用户趋势

```sql
SELECT
    stat_date,
    COUNT(DISTINCT user_id) as dau,
    SUM(call_count) as total_calls,
    ROUND(AVG(call_count), 2) as avg_calls_per_user
FROM dws_user_daily
GROUP BY stat_date
ORDER BY stat_date DESC
LIMIT 30;
```

### 5.2 模型使用趋势

```sql
SELECT
    stat_date,
    model_name,
    call_count,
    user_count,
    avg_tokens_per_call
FROM dws_model_daily
WHERE model_name IN ('claude-sonnet-4-6', 'claude-haiku-4-5-20251001')
ORDER BY stat_date DESC, call_count DESC;
```

### 5.3 用户留存分析

```sql
SELECT
    user_segment,
    COUNT(*) as user_count,
    SUM(CASE WHEN is_active_7d = 1 THEN 1 ELSE 0 END) as active_7d,
    SUM(CASE WHEN is_active_30d = 1 THEN 1 ELSE 0 END) as active_30d,
    ROUND(AVG(days_since_first), 1) as avg_days_since_first
FROM dws_user_lifecycle
GROUP BY user_segment
ORDER BY user_segment;
```

### 5.4 用户价值分析

```sql
SELECT
    user_id,
    total_call_count,
    total_topup_amount,
    user_segment,
    ROUND(total_topup_amount / NULLIF(total_call_count, 0), 4) as revenue_per_call
FROM dws_user_lifecycle
WHERE total_topup_amount > 0
ORDER BY total_topup_amount DESC
LIMIT 20;
```
