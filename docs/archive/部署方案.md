# 数仓部署方案

## 📋 部署策略选择

根据您的需求，有两种部署方案：

---

## 方案1: 本地同步后上传（推荐）

### 适用场景
- ✅ 本地网络稳定，可以完成全量同步
- ✅ 希望减少服务器压力
- ✅ 首次部署时使用

### 操作步骤

#### 1. 本地完成全量同步
```bash
# 在本地运行全量同步
python sync_full.py
```

**预计时间**: 2-3小时（47万+条数据）

#### 2. 上传到服务器
```bash
# 需要上传的文件
- newapi_warehouse.db          # SQLite数据库文件（约800MB-1GB）
- sync_full.py                 # 全量同步脚本（备用）
- sync_incremental.py          # 增量同步脚本（后续创建）
- database.py                  # 数据库操作模块
- api_client.py                # API客户端
- utils.py                     # 工具函数
- config.py                    # 配置文件（后续创建）
```

#### 3. 服务器上设置增量同步
```bash
# 创建定时任务（每日凌晨2点同步前一天数据）
# 使用cron或systemd timer
```

### 优点
- ✅ 减少服务器首次同步压力
- ✅ 本地网络通常更稳定
- ✅ 可以验证数据完整性后再上传

### 缺点
- ⚠️ 需要上传较大的数据库文件（约1GB）
- ⚠️ 上传时间可能较长

---

## 方案2: 服务器直接同步

### 适用场景
- ✅ 服务器网络稳定
- ✅ 不便于上传大文件
- ✅ 希望所有操作在服务器完成

### 操作步骤

#### 1. 上传代码到服务器
```bash
# 只上传代码文件，不包含数据库
- sync_full.py
- sync_incremental.py
- database.py
- api_client.py
- utils.py
- config.py
```

#### 2. 服务器上初始化数据库
```bash
# 创建空数据库和表结构
python database.py
```

#### 3. 服务器上运行全量同步
```bash
# 运行全量同步（预计2-3小时）
python sync_full.py
```

#### 4. 设置增量同步定时任务

### 优点
- ✅ 不需要上传大文件
- ✅ 所有数据在服务器本地
- ✅ 部署流程简单

### 缺点
- ⚠️ 首次同步占用服务器资源
- ⚠️ 需要确保服务器网络稳定

---

## 🎯 推荐方案：混合方案

### 最佳实践

**首次部署**：使用方案1（本地同步后上传）
- 本地完成全量同步
- 验证数据完整性
- 上传数据库文件到服务器

**后续更新**：使用增量同步
- 服务器每日自动增量同步
- 只同步新数据（created_at > 上次最大值）

## 🔄 增量同步设置

### 创建增量同步脚本

需要创建一个 `sync_incremental.py`，功能：
1. 读取上次同步的最大 `created_at`
2. 只同步新数据（`created_at > 上次最大值`）
3. 更新同步记录

### 定时任务配置

#### Linux Cron方式
```bash
# 编辑crontab
crontab -e

# 添加每日凌晨2点同步任务
0 2 * * * cd /path/to/newapi_export && /usr/bin/python3 sync_incremental.py >> /var/log/newapi_sync.log 2>&1
```

#### Systemd Timer方式（推荐）
```ini
# /etc/systemd/system/newapi-sync.service
[Unit]
Description=NewAPI Data Sync
After=network.target

[Service]
Type=oneshot
User=your_user
WorkingDirectory=/path/to/newapi_export
ExecStart=/usr/bin/python3 sync_incremental.py
StandardOutput=journal
StandardError=journal

# /etc/systemd/system/newapi-sync.timer
[Unit]
Description=NewAPI Daily Sync Timer

[Timer]
OnCalendar=daily
OnCalendar=02:00
Persistent=true

[Install]
WantedBy=timers.target
```

## ❓ 常见问题

### Q1: 上传数据库文件需要多长时间？
**A**: 取决于网络速度。1GB文件：
- 10Mbps: 约15分钟
- 100Mbps: 约1.5分钟
- 1Gbps: 约10秒

### Q2: 上传期间有新数据怎么办？
**A**: 不用担心，增量同步会自动补全。增量同步会从上次同步的最大 `created_at` 开始，自动获取所有新数据。

### Q3: 数据库文件损坏怎么办？
**A**: SQLite有WAL模式保护，但建议：
- 定期备份数据库文件
- 使用 `VACUUM` 命令优化数据库
- 保留同步记录，可以重新同步

### Q4: 如何验证上传后的数据完整性？
**A**: 
```python
from database import Database

db = Database()
db.connect()

# 检查记录数
cursor = db.execute("SELECT COUNT(*) as cnt FROM ods_logs")
print(f"日志记录数: {cursor.fetchone()['cnt']:,}")

# 检查时间范围
cursor = db.execute("SELECT MIN(created_at) as min_t, MAX(created_at) as max_t FROM ods_logs")
row = cursor.fetchone()
print(f"时间范围: {row['min_t']} ~ {row['max_t']}")

db.close()
```

**建议**: 先在本地完成全量同步，验证无误后再上传到服务器。
