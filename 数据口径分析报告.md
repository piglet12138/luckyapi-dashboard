## 一、留存率趋势问题

### 1.1 当前数据情况

```sql
-- 最近10天的留存率数据
stat_date   | retention_rate | conversion_rate | repurchase_rate
------------|----------------|-----------------|----------------
2026-03-14  | 1.41          | 435.94          | 39.43
2026-03-13  | 5.48          | 204.55          | 40.37
2026-03-12  | 5.88          | 220.35          | 41.37
```

### 1.2 数据口径分析

**retention_rate（7日留存率）计算逻辑**:

```sql
-- 分子：7天前新增且今天仍活跃的用户数
SELECT COUNT(*) FROM dws_user_lifecycle
WHERE first_use_date = date(stat_date, '-7 days')
  AND last_use_date >= stat_date

-- 分母：7天前的活跃用户数
SELECT COUNT(DISTINCT user_id) FROM dws_user_daily
WHERE stat_date = date(stat_date, '-7 days')
```

### 1.3 问题诊断

**问题1：计算逻辑错误** ❌

当前逻辑：
- 分子：7天前**首次使用**且今天仍活跃的用户
- 分母：7天前的**活跃用户**

这个逻辑有问题：
- 分子只统计7天前的**新用户**，不包括老用户
- 分母统计7天前的**所有活跃用户**（包括新老用户）
- 导致留存率被严重低估

**正确的7日留存率应该是**：
- 分子：7天前活跃的用户中，今天仍活跃的用户数
- 分母：7天前活跃的用户数

**问题2：conversion_rate异常** ❌

```sql
conversion_rate = 累计付费用户 / 当日活跃用户 * 100
```

这个计算有严重问题：
- 分子是**累计付费用户**（历史所有付费用户）
- 分母是**当日活跃用户**
- 导致转化率>100%（如435.94%）

**正确的付费转化率应该是**：
- 当日转化率：当日付费用户 / 当日活跃用户
- 或累计转化率：累计付费用户 / 累计注册用户

### 1.4 数据验证

```bash
# 验证数据异常
sqlite3 newapi_warehouse.db "
SELECT
    stat_date,
    active_users,
    paying_users,
    conversion_rate,
    ROUND(CAST(paying_users AS FLOAT) / active_users * 100, 2) as correct_rate
FROM ads_funnel_daily
ORDER BY stat_date DESC
LIMIT 5
"
```

## 二、渠道转化率对比问题

### 2.1 当前数据情况

```sql
-- 渠道转化率数据
channel                              | new_users | paying_users | avg_conversion_rate
-------------------------------------|-----------|--------------|--------------------
claude-sonnet-4-6                    | 171       | 94           | 54.29
gcp/gemini-3-pro-preview-thinking    | 14        | 14           | 100.0
```

### 2.2 数据口径分析

**channel_details导出逻辑**:

```sql
SELECT
    first_model as channel,
    SUM(new_user_count) as new_users,
    SUM(paying_user_count) as paying_users,
    ROUND(AVG(conversion_rate), 2) as conversion_rate
FROM ads_channel_attribution
GROUP BY first_model
```

**ads_channel_attribution表结构**:
- 按**日期 + 渠道**维度存储
- 每条记录是某天某渠道的新增用户和付费情况

### 2.3 问题诊断

**问题：时间范围不明确** ⚠️

- 数据是**历史累计**（所有日期的汇总）
- 但看板上没有说明时间范围
- 用户无法判断是当天、本周还是全部历史

**问题：AVG(conversion_rate)不准确** ⚠️

```sql
-- 当前计算：对每天的转化率求平均
AVG(conversion_rate)

-- 正确计算：总付费用户 / 总新增用户
SUM(paying_user_count) / SUM(new_user_count) * 100
```

使用AVG会导致：
- 小样本日期的转化率权重过大
- 不能真实反映渠道的整体转化率

### 2.4 数据验证

```bash
# 验证转化率计算
sqlite3 newapi_warehouse.db "
SELECT
    first_model,
    SUM(new_user_count) as new_users,
    SUM(paying_user_count) as paying_users,
    ROUND(AVG(conversion_rate), 2) as avg_rate,
    ROUND(CAST(SUM(paying_user_count) AS FLOAT) / SUM(new_user_count) * 100, 2) as correct_rate
FROM ads_channel_attribution
GROUP BY first_model
ORDER BY new_users DESC
LIMIT 5
"
```

## 五、修复建议

### 5.1 高优先级修复（数据错误）

**1. 修复留存率计算逻辑**

```sql
-- 正确的7日留存率
retention_rate =
    (7天前活跃且今天仍活跃的用户数) / (7天前活跃用户数) * 100

-- SQL实现
SELECT
    COUNT(DISTINCT CASE WHEN last_use_date >= stat_date THEN user_id END) /
    COUNT(DISTINCT user_id) * 100
FROM dws_user_daily
WHERE stat_date = date('now', '-7 days')
```

**2. 修复付费转化率计算逻辑**

```sql
-- 当日转化率（推荐）
conversion_rate = 当日付费用户 / 当日活跃用户 * 100

-- 或累计转化率
conversion_rate = 累计付费用户 / 累计注册用户 * 100
```

### 5.2 中优先级修复（数据不准确）

**3. 修复渠道转化率计算**

```sql
-- 从AVG改为加权平均

    SUM(paying_user_count) / SUM(new_user_count) * 100
```

## 六、验证SQL

### 6.1 验证留存率问题

```sql
-- 查看2026-03-14的留存率计算
SELECT
    '2026-03-14' as stat_date,

    -- 7天前（2026-03-07）活跃用户数
    (SELECT COUNT(DISTINCT user_id)
     FROM dws_user_daily
     WHERE stat_date = '2026-03-07') as users_7d_ago,

    -- 7天前活跃且今天仍活跃的用户数
    (SELECT COUNT(DISTINCT d1.user_id)
     FROM dws_user_daily d1
     WHERE d1.stat_date = '2026-03-07'
       AND EXISTS (
           SELECT 1 FROM dws_user_daily d2
           WHERE d2.user_id = d1.user_id
             AND d2.stat_date = '2026-03-14'
       )) as retained_users,

    -- 正确的留存率
    ROUND(
        CAST((SELECT COUNT(DISTINCT d1.user_id)
              FROM dws_user_daily d1
              WHERE d1.stat_date = '2026-03-07'
                AND EXISTS (
                    SELECT 1 FROM dws_user_daily d2
                    WHERE d2.user_id = d1.user_id
                      AND d2.stat_date = '2026-03-14'
                )) AS FLOAT) /
        NULLIF((SELECT COUNT(DISTINCT user_id)
                FROM dws_user_daily
                WHERE stat_date = '2026-03-07'), 0) * 100,
        2
    ) as correct_retention_rate,

    -- 当前错误的留存率
    (SELECT retention_rate
     FROM ads_funnel_daily
     WHERE stat_date = '2026-03-14') as current_retention_rate
;
```

### 6.2 验证转化率问题

```sql
-- 查看2026-03-14的转化率
SELECT
    stat_date,
    active_users,

    -- 当日付费用户
    (SELECT COUNT(DISTINCT user_id)
     FROM dwd_logs
     WHERE log_type = 1 AND log_date = '2026-03-14') as daily_paying_users,

    -- 累计付费用户
    paying_users as cumulative_paying_users,

    -- 正确的当日转化率
    ROUND(
        CAST((SELECT COUNT(DISTINCT user_id)
              FROM dwd_logs
              WHERE log_type = 1 AND log_date = '2026-03-14') AS FLOAT) /
        active_users * 100,
        2
    ) as correct_daily_conversion_rate,

    -- 当前错误的转化率
    conversion_rate as current_conversion_rate
FROM ads_funnel_daily
WHERE stat_date = '2026-03-14'
;
```

### 6.3 验证渠道转化率

```sql
-- 对比AVG和加权平均
SELECT
    first_model,
    SUM(new_user_count) as total_new_users,
    SUM(paying_user_count) as total_paying_users,

    -- 当前方法：AVG
    ROUND(AVG(conversion_rate), 2) as avg_conversion_rate,

    -- 正确方法：加权平均
    ROUND(
        CAST(SUM(paying_user_count) AS FLOAT) /
        SUM(new_user_count) * 100,
        2
    ) as weighted_conversion_rate,

    -- 差异
    ROUND(
        CAST(SUM(paying_user_count) AS FLOAT) /
        SUM(new_user_count) * 100 -
        AVG(conversion_rate),
        2
    ) as difference
FROM ads_channel_attribution
GROUP BY first_model
HAVING SUM(new_user_count) >= 10  -- 过滤小样本
ORDER BY total_new_users DESC
LIMIT 10
;
```
