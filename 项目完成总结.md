## 📋 项目概览

本项目是一个完整的数据分析看板系统，包含：
- **数据采集**: 从NewAPI自动同步数据
- **数据仓库**: 四层架构（ODS → DWD → DWS → ADS）
- **数据看板**: 可视化展示核心业务指标
- **自动更新**: 每日增量更新机制
- **服务器部署**: 完整的部署方案

## 🚀 快速开始

### 1. 本地测试

```bash
# 进入项目目录
cd /path/to/newapi_export

# 测试增量更新
python daily_update.py

# 启动看板（本地）
cd dashboard
python -m http.server 8000
# 访问 http://localhost:8000
```

### 2. 配置定时任务

**Linux/Mac (crontab):**
```bash
crontab -e

# 每天凌晨2点执行
0 2 * * * cd /opt/newapi-dashboard && python3 daily_update.py >> logs/cron.log 2>&1

# 每天凌晨3点备份
0 3 * * * /opt/newapi-dashboard/deploy/backup.sh
```

**Windows (任务计划程序):**
```powershell
# 创建定时任务
$action = New-ScheduledTaskAction -Execute "python" -Argument "C:\newapi-dashboard\daily_update.py"
$trigger = New-ScheduledTaskTrigger -Daily -At 2am
Register-ScheduledTask -TaskName "NewAPI Daily Update" -Action $action -Trigger $trigger
```

### 3. 服务器部署

**方式1: 一键部署（推荐）**
```bash
cd /opt/newapi-dashboard
chmod +x deploy.sh
./deploy.sh
```

**方式2: Docker Compose**
```bash
cd /opt/newapi-dashboard
cp .env.example .env
# 编辑 .env 配置
vim .env

docker-compose up -d
```

**方式3: Nginx直接部署**
```bash
# 安装依赖
sudo apt-get install -y nginx python3-pip
pip3 install -r requirements.txt

# 配置Nginx
sudo cp deploy/nginx.conf /etc/nginx/sites-available/newapi-dashboard
sudo ln -s /etc/nginx/sites-available/newapi-dashboard /etc/nginx/sites-enabled/
sudo systemctl reload nginx

# 配置定时任务
crontab -e
```

## 🔄 每日更新流程

**执行命令**: `python daily_update.py`

**完整流程** (约6分钟):

```
1. 增量同步API数据 (~5-6分钟)
   ├─ 检查数据库最新记录
   ├─ 拉取新增日志 (12,000+ 条/天)
   ├─ 全量更新用户快照 (1,200+ 用户)
   └─ 全量更新渠道快照 (30+ 渠道)

2. ODS → DWD 数据清洗 (~30秒)
   ├─ 转换新日志数据
   ├─ 清洗字段、脱敏处理
   └─ 转换用户数据

3. DWD → DWS 数据汇总 (~2秒)
   ├─ 汇总用户每日数据 (最近7天)
   ├─ 汇总模型每日数据 (最近7天)
   ├─ 汇总渠道每日数据 (最近7天)
   └─ 更新用户生命周期 (全量)

4. DWS → ADS 指标计算 (~2秒)
   ├─ 计算每日核心指标 (最近30天)
   ├─ 计算转化漏斗 (最近30天)
   ├─ 计算用户分层 (最近30天)
   ├─ 计算渠道归因 (最近30天)
   ├─ 计算新用户转化率 (最近30天)
   ├─ 计算激活点分析 (全量)
   └─ 计算分层复购率 (全量)

5. 导出看板数据 (~1秒)
   └─ 导出到 dashboard/dashboard_data.json
```

6. **激活点分析** - 不同激活点的用户转化

**复购分析** (Phase 2):

## 🔧 故障排查

### 问题1: 增量更新失败

**症状**: `daily_update.py` 执行报错

**排查步骤**:
```bash
# 1. 查看日志
tail -100 logs/daily_update.log

# 2. 检查API连接
python -c "from src.core.api_client import NewAPIClient; from src.core.config import BASE_URL, TOKEN; client = NewAPIClient(BASE_URL, TOKEN); print(client.get_logs(page=1, page_size=1))"

# 3. 检查数据库
sqlite3 newapi_warehouse.db ".tables"

# 4. 手动执行测试
python daily_update.py
```

### 问题2: 看板无数据

**症状**: 看板显示空白或数据不更新

**排查步骤**:
```bash
# 1. 检查JSON文件
ls -lh dashboard/dashboard_data.json
python -c "import json; print(json.load(open('dashboard/dashboard_data.json', encoding='utf-8'))['update_time'])"

# 2. 检查数据库数据
sqlite3 newapi_warehouse.db "SELECT MAX(stat_date) FROM ads_daily_summary;"

# 3. 重新导出数据
python -c "from export_dashboard_data import DashboardDataExporter; exporter = DashboardDataExporter(); exporter.export_all()"

# 4. 清除浏览器缓存后重新访问
```

### 问题3: 定时任务未执行

**症状**: 数据长时间未更新

**排查步骤**:
