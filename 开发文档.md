## 1. 项目概述

### 1.1 项目背景

NewAPI 数据仓库项目旨在为 NewAPI 服务构建完整的数据分析体系，支持业务指标监控、用户行为分析和渠道效果评估。

### 1.2 核心目标

- 建立标准化的数据仓库分层架构（ODS → DWD → DWS → ADS）
- 实现数据脱敏和隐私保护
- 提供业务指标计算和分析能力
- 支持数据可视化看板

### 1.3 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 数据库 | SQLite | 轻量级、无需部署 |
| 开发语言 | Python 3.8+ | 数据处理和ETL |
| 前端框架 | Bootstrap 5 + ECharts | 数据可视化 |
| 数据格式 | JSON | 前后端数据交换 |

## 3. 数据仓库设计

### 3.1 分层架构

#### ODS层（Operational Data Store）

**目的**: 存储原始数据，保持与源系统一致

**表结构**: `ods_logs`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| log_type | INTEGER | 日志类型（1=充值，2=调用） |
| username | TEXT | 用户名（原始） |
| token_name | TEXT | Token名称 |
| model_name | TEXT | 模型名称 |
| quota | INTEGER | 配额 |
| prompt_tokens | INTEGER | 提示词Token数 |
| completion_tokens | INTEGER | 完成Token数 |
| content | TEXT | 日志内容（JSON） |
| created_at | INTEGER | 创建时间戳 |

#### DWD层（Data Warehouse Detail）

**目的**: 清洗、脱敏后的明细数据

**表结构**: `dwd_logs`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| log_type | INTEGER | 日志类型 |
| user_id | TEXT | 用户ID（脱敏） |
| email | TEXT | 邮箱（脱敏） |
| ip_address | TEXT | IP地址（脱敏） |
| model_name | TEXT | 模型名称 |
| quota | INTEGER | 配额 |
| prompt_tokens | INTEGER | 提示词Token数 |
| completion_tokens | INTEGER | 完成Token数 |
| topup_amount | REAL | 充值金额 |
| log_date | TEXT | 日志日期 |
| log_hour | INTEGER | 日志小时 |
| log_weekday | INTEGER | 星期几 |
| created_at | INTEGER | 创建时间戳 |

**脱敏规则**:
- 用户名: `user_1007` 格式
- 邮箱: `user_1007@domain.com` 格式
- IP地址: `192.168.*.*` 格式

#### DWS层（Data Warehouse Summary）

**目的**: 按维度汇总的数据

**主要表**:

1. `dws_user_daily` - 用户每日汇总
2. `dws_user_lifecycle` - 用户生命周期
3. `dws_model_daily` - 模型每日汇总

#### ADS层（Application Data Store）

**目的**: 面向业务的应用数据

**主要表**:

1. `ads_daily_summary` - 每日核心指标
2. `ads_funnel_daily` - 增长漏斗
3. `ads_user_segment_daily` - 用户分层
4. `ads_activation_analysis` - 激活点分析
5. `ads_channel_attribution` - 渠道归因

### 3.2 数据字典

详细的字段说明请参考 `数据字典.md`

## 6. ETL流程

### 6.1 数据流转

```
原始数据 → ODS层 → DWD层 → DWS层 → ADS层 → 看板
```

### 6.2 执行顺序

1. **ODS层**: 从源数据库导入原始数据
2. **DWD层**: 数据清洗和脱敏
3. **DWS层**: 按维度汇总
4. **ADS层**: 计算业务指标
5. **导出**: 生成JSON数据供看板使用

### 6.3 性能优化

**SQL优化技巧**:

```sql
-- 使用临时表避免相关子查询
CREATE TEMP TABLE user_stats AS
SELECT user_id, COUNT(*) as call_count
FROM dwd_logs
WHERE log_type = 2
GROUP BY user_id;

-- 使用索引加速查询
CREATE INDEX idx_user_date ON dws_user_daily(user_id, stat_date);
```

**批量处理**:

```python
# 批量插入而非逐条插入
cursor.executemany(
    "INSERT INTO dwd_logs VALUES (?, ?, ?)",
    batch_data
)
```

## 8. 扩展开发

### 8.1 添加新指标

**步骤**:

1. 在ADS层创建新表或添加新字段
2. 在 `src/etl/dws_to_ads.py` 中添加计算逻辑
3. 在 `export_dashboard_data.py` 中添加导出函数
4. 在看板中添加可视化

**示例 - 添加用户留存率**:

```python
# 1. 在 dws_to_ads.py 中添加计算函数
def calculate_retention_rate(self):
    sql = """
    INSERT INTO ads_retention_analysis
    SELECT
        stat_date,
        COUNT(DISTINCT user_id) as retained_users,
        retention_rate
    FROM ...
    """
    self.db.execute(sql)

# 2. 在 export_dashboard_data.py 中添加导出
def export_retention_analysis(self):
    cursor = self.conn.execute("""
        SELECT * FROM ads_retention_analysis
        ORDER BY stat_date DESC
        LIMIT 30
    """)
    return [self._row_to_dict(row) for row in cursor.fetchall()]
```

### 8.2 增量更新

**设计思路**:

def incremental_update(self, start_date: str):
    """增量更新指定日期之后的数据"""
    # 1. 删除旧数据
    self.db.execute("DELETE FROM ads_daily_summary WHERE stat_date >= ?", (start_date,))

### 8.3 数据质量监控

**关键指标**:

- 数据完整性：各层记录数是否一致
- 数据准确性：关键指标是否合理
- 数据时效性：最新数据日期

**监控脚本**:

```python
def check_data_quality():
    # 检查记录数
    ods_count = db.execute("SELECT COUNT(*) FROM ods_logs").fetchone()[0]
    dwd_count = db.execute("SELECT COUNT(*) FROM dwd_logs").fetchone()[0]

    if ods_count != dwd_count:
        alert("数据完整性异常")

    # 检查最新日期
    latest_date = db.execute("SELECT MAX(stat_date) FROM ads_daily_summary").fetchone()[0]
    if latest_date < today - 1:
        alert("数据时效性异常")
```

## 9. 常见问题

### 9.1 数据库锁定

**问题**: `database is locked`

**解决**:
```python
# 设置超时时间
conn = sqlite3.connect("db.db", timeout=30)

# 或使用WAL模式
conn.execute("PRAGMA journal_mode=WAL")
```

### 9.2 内存不足

**问题**: 大数据量处理时内存溢出

**解决**:
```python
# 分批处理
batch_size = 10000
for offset in range(0, total_count, batch_size):
    batch = fetch_batch(offset, batch_size)
    process_batch(batch)
```

### 9.3 SQL性能慢

**问题**: 查询执行时间过长

**解决**:
```sql
-- 添加索引
CREATE INDEX idx_log_date ON dwd_logs(log_date);
CREATE INDEX idx_user_id ON dwd_logs(user_id);

-- 使用EXPLAIN分析
EXPLAIN QUERY PLAN SELECT ...;
```

### 9.4 字符编码问题

**问题**: Windows下中文乱码

**解决**:
```python
# 统一使用UTF-8
with open(file, 'r', encoding='utf-8') as f:
    content = f.read()
```
